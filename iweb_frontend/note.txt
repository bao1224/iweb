111、面试题：大型页面时CSS的章法？
  一个大型页面（例如：京东首页）CSS大约有1000行，编写时有一定的章法：
  ①Reset：样式重置，把HTML标签的默认样式统一化——可以使用Normalize.css
  ②Scaffolding:样式脚手架，设定全局的基础样式
  ③Layout System：布局系统，进行页面布局必需的class
  ④Components:组件，多个页面所共用的通用组件必需的样式
  
  
  
2、使用浮动布局时常见的问题
   若容器中的元素浮动起来，则不再占用布局空间;
   如果容器内所有的子元素全部浮动，则父容器不再有高度。
   解决方案：给父容器添加一个新的没有浮动的子元素
   .col {
	   float:after;
   }
   .row:after {     /*在父容器尾部生成新孩子*/
	   content:'';
	   display:block;
	   clear:both;     /*清楚兄弟元素浮动造成的影响*/
   }
   
   3、项目中大量小图标的处理方式
     方式1：每个小图片都是一个独立的.png文件，例如：
	 search.png、user.png、star.png...HTML中使用<img>显示图片。
	 不足：会增加浏览器资源请求次数，导致页面加载效率变低。
	 方式2：把多个小图片拼接为一个大的，png图片（称为“Sprites Image"(精灵图/雪碧图)），
	 使用时利用浏览器的缓存机制（请求一次之后，再次用到该图片无需再请求）可以
	 实现大图片仅请求一次的效果。
	 不足：如果需要添加新的图标必需修改png图片：因为使用使用时必须配合
	 CSS Background属性来使用，有些场合下不允许使用。
	 方式3、使用图标字体（icon font），本质是文字字符（可以无限
	 缩放/改变颜色/改变背景/加粗/变斜），呈现的外观是图标。——推荐使用
	 开源的FontAwesome图标字体（提供了675个常用图标）。
